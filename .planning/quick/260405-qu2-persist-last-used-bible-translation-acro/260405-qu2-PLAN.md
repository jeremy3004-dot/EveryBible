<objective>
Fix the bug where the app resets currentTranslation to BSB on every restart when the user's last-used translation is a downloaded runtime translation (e.g., hincv, sparv1909, npiulb, or any Phase 28 Supabase translation).

Purpose: The first loop in mergeRuntimeCatalogTranslations drops ALL runtime translations from state, including ones that are already downloaded locally. When applyRuntimeCatalog re-merges them from the Supabase catalog response, any translation missing from that response disappears from the ID set, causing the fallback to 'bsb'.

Output: One-line condition change in bibleStoreModel.ts, plus a regression test.
</objective>
