# Plan 02: FlashList Upgrades On Heavy Bible Surfaces

## Goal

Improve responsiveness on the heaviest Bible list surfaces without changing navigation, reader behavior, or audio-related contracts.

## Scope

- Add `@shopify/flash-list` as the list virtualization dependency.
- Migrate the simplest high-value Bible surfaces first:
  - `ChapterSelectorScreen`
  - `BibleBrowserScreen` browse/search results
- Preserve existing row visuals, tap targets, and navigation semantics.

## Files

- `package.json`
- `src/screens/bible/BibleBrowserScreen.tsx`
- `src/screens/bible/ChapterSelectorScreen.tsx`

## Tasks

1. Add failing coverage or deterministic checks for any list-specific helper behavior touched by the migration.
2. Replace current `FlatList` usage with FlashList in the selected surfaces using conservative estimated sizes.
3. Re-run parser/search behavior after virtualization changes so list upgrades do not regress scripture entry or navigation.

## Non-Goals

- Reworking `BibleReaderScreen` verse rendering in this phase
- Rebuilding modal sheets that do not have meaningful list pressure
- Visual redesign of Bible browsing
