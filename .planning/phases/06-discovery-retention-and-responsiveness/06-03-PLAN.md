# Plan 03: Reading Activity Calendar

## Goal

Show users their recent reading momentum in a calendar view backed by current progress state and reachable from the More/Profile surface.

## Scope

- Add `react-native-calendars` as the calendar dependency.
- Add a small reading-activity service that converts stored reading progress into calendar markings and selected-day summaries.
- Add a `ReadingActivityScreen` in the More stack and a clear entry point from `ProfileScreen`.
- Keep the feature read-only and focused on activity, streak, and chapter counts.

## Files

- `package.json`
- `src/navigation/MoreStack.tsx`
- `src/navigation/types.ts`
- `src/screens/more/ProfileScreen.tsx`
- `src/screens/more/ReadingActivityScreen.tsx`
- `src/services/progress/readingActivity.ts`
- `src/services/progress/readingActivity.test.ts`
- `src/stores/progressStore.ts`
- `src/i18n/locales/en.ts`

## Tasks

1. Add failing tests for turning stored reading data into per-day calendar markings and summaries.
2. Backfill the smallest reliable local activity model needed for the calendar while preserving current streak/stat behavior.
3. Add the screen, route, and Profile entry point using a straightforward month-view experience.

## Non-Goals

- Full reading-plan authoring or assignment
- Reminder scheduling changes
- Mandatory sync of historical activity if local-first support already satisfies the screen
