# Plan 01: Typed Passage Parsing And Direct Navigation

## Goal

Let users type common scripture references into the existing Bible search field and jump directly to the intended passage while preserving full-text search behavior.

## Scope

- Add `bible-passage-reference-parser` as the parsing dependency.
- Add a normalization layer that maps parser output into app `bookId`, `chapter`, and optional `focusVerse`.
- Extend the Bible search model so the screen can distinguish:
  - parsable reference input
  - normal full-text search
  - too-short/no-op input
- Wire direct navigation into `BibleBrowserScreen` without creating a new screen or route shape.

## Files

- `package.json`
- `src/constants/books.ts`
- `src/screens/bible/BibleBrowserScreen.tsx`
- `src/screens/bible/bibleSearchModel.ts`
- `src/screens/bible/bibleSearchModel.test.ts`
- `src/services/bible/referenceParser.ts`
- `src/services/bible/referenceParser.test.ts`

## Tasks

1. Add failing tests for reference parsing, mapping, and search-model classification.
2. Implement the smallest parser adapter that supports common English book names and abbreviations mapped to current app IDs.
3. Update `BibleBrowserScreen` to route parsed references straight into `BibleReader` while keeping plain-text search unchanged.

## Non-Goals

- Localized book-name parsing beyond current English book metadata
- Multi-chapter reading routes
- Deep-link routing expansion unless required by tests
