<objective>
Add push token registration to user_devices on sign-in, build the send-group-notification Edge Function for group session alerts, and wire group session creation to trigger push notifications to group members.

Purpose: Push tokens are the bridge between the app and server-initiated notifications. Without token registration, the server cannot reach devices. The Edge Function enables group leaders to notify all members when a session is recorded.

Output: registerPushToken + deactivatePushToken in notificationService, send-group-notification Edge Function, group session alert trigger in groupService.

       ```typescript
       export async function registerPushToken(userId: string): Promise<string | null> {
         try {
           const projectId = Constants.expoConfig?.extra?.eas?.projectId as string;
           if (!projectId) return null;

           const { status } = await Notifications.getPermissionsAsync();
           if (status !== 'granted') return null;

           const tokenResult = await Notifications.getExpoPushTokenAsync({ projectId });
           cachedPushToken = tokenResult.data;

           const platform: 'ios' | 'android' = Platform.OS === 'ios' ? 'ios' : 'android';
           await supabase.from('user_devices').upsert(
             {
               user_id: userId,
               push_token: tokenResult.data,
               platform,
               is_active: true,
               updated_at: new Date().toISOString(),
             },
             { onConflict: 'user_id,push_token' }
           );

           return tokenResult.data;
         } catch {
           // Non-fatal: simulator, offline, or RLS error — do not crash
           return null;
         }
       }
       ```

    4. Add `deactivatePushToken(userId: string)`:
       ```typescript
       export async function deactivatePushToken(userId: string): Promise<void> {
         try {
           if (!cachedPushToken) return;
           await supabase
             .from('user_devices')
             .update({ is_active: false, updated_at: new Date().toISOString() })
             .eq('user_id', userId)
             .eq('push_token', cachedPushToken);
           cachedPushToken = null;
         } catch {
           // Non-fatal
         }
       }
       ```

    **Create `supabase/functions/send-group-notification/index.ts`:**

    Follow the exact pattern from `aggregate-engagement/index.ts`:
    - Import `createClient` from `https://esm.sh/@supabase/supabase-js@2`
    - Add CORS headers (same as aggregate-engagement)
    - Handle OPTIONS preflight
    - Create Supabase client with `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` env vars

    Request body: `{ group_id: string, title: string, body: string, exclude_user_id?: string }`
    The `exclude_user_id` is the session creator (leader) — they should not receive a notification about their own action.

    Logic:
    1. Query `group_members` to get all `user_id` values for the given `group_id`.
    2. Filter out `exclude_user_id` if provided.
    3. Query `user_devices` for active tokens: `.select('push_token').eq('is_active', true).in('user_id', memberUserIds)`.
    4. If no tokens, return `{ success: true, sent: 0, reason: 'no_active_tokens' }`.
    5. Build Expo Push API message array: `devices.map(d => ({ to: d.push_token, title, body, sound: 'default', data: { screen: 'GroupDetail', groupId: group_id } }))`.
    6. Batch into chunks of 100 (Expo limit per request).
    7. POST each batch to `https://exp.host/--/api/v2/push/send` with `Content-Type: application/json` and `Accept: application/json`.
    8. Collect results and return `{ success: true, sent: totalSent, errors: totalErrors }`.

    Error handling: wrap everything in try/catch, return 500 with error message on failure.

    **Wire in group session creation:**
