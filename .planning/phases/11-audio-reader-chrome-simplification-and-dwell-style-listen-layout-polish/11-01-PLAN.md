# Plan 11-01: Simplify Audio-first Reader Chrome

## Objective

Strip the audio-first Bible screen down to the core top chrome the user actually wants: back, `Listen / Read`, and overflow only.

## Scope

- Add an audio-first-specific header treatment in `src/screens/bible/BibleReaderScreen.tsx`.
- Remove the top chapter title, translation chip, and `AA` button for audio-first chapters.

- Keep overflow actions and the `Listen / Read` session rail intact.

## Out Of Scope

- Changing translation-selection behavior for text chapters
- Redesigning the overflow action sheet contents
