# Plan 18-04: Plan Detail Screen Enhancement

## Goal

When this plan is complete, the PlanDetailScreen shows a full-width cover image header, completion count badge, a CTA row with "Start Plan" / "Save For Later" / "Sample" buttons, plan description, the existing day-list with mark-complete functionality, and a "Related Plans" horizontal scroll section at the bottom.

## Scope

This plan does:
- Enhance `src/screens/plans/PlanDetailScreen.tsx` (or the underlying `src/screens/learn/ReadingPlanDetailScreen.tsx`) with:
  - Full-width cover image header (height ~220, with gradient overlay at bottom for text readability)
  - Completion count label (e.g., "5,280 completions") below the image
  - CTA row: "Start Plan" (primary button, only if not enrolled), "Save For Later" (secondary, toggles to "Unsaved" if already saved), "Sample" (tertiary, scrolls to day 1 entry)
  - Plan description text block
  - The existing day-list and mark-complete behavior preserved as-is
  - "Related Plans" horizontal scroll section at the bottom (same category, limit 5, excludes current plan)
- Wire the "Save For Later" button to `savePlanForLater` / `unsavePlan` service functions
- Wire the "Start Plan" button to the existing `enrollInPlan` service function
- Add related plans query using `getPlansByCategory`

This plan does NOT:
- Modify the PlansHomeScreen (Plan 18-03)
- Touch navigation wiring or types (Plan 18-02)
- Add star ratings, audio plans, group assignment UI, or push reminders (deferred)
- Re-implement any existing day-list or mark-complete logic

## Prerequisites

- Plan 18-01 must be complete (service functions, types with cover_image_url/featured/completion_count)
- Plan 18-02 must be complete (PlanDetailScreen wrapper, navigation types)
- Plan 18-03 should ideally be complete but is not strictly required (the detail screen works independently)

### Task 2: Add CTA row with Start Plan, Save For Later, and Sample

Below the title section, add a horizontal row of 3 buttons:

**"Start Plan" button (primary):**
- Only visible if user is NOT already enrolled in this plan
- If enrolled: show "Continue" instead, or hide entirely (the day-list already has mark-complete buttons)
- Style: `backgroundColor: colors.accentPrimary`, `color: colors.cardBackground`, full rounded pill, bold text
- onPress: call `enrollInPlan(planId)`, then refresh the screen data
- If user is not authenticated: call `openAuthFlow('SignIn')` from `rootNavigation.ts` and return

**"Save For Later" button (secondary):**
- Check if plan is saved: on mount, query `getSavedPlans()` and check if current planId is in the list (or add a local state flag)
- Not saved: show `t('readingPlans.saveForLater')` with Ionicons `bookmark-outline`
- Already saved: show `t('readingPlans.unsave')` with Ionicons `bookmark` (filled)
- onPress: toggle — call `savePlanForLater(planId)` or `unsavePlan(planId)`
- Style: `borderWidth: 1`, `borderColor: colors.cardBorder`, `backgroundColor: 'transparent'`, text `colors.primaryText`, rounded pill
- If user is not authenticated: call `openAuthFlow('SignIn')`

**"Sample" button (tertiary):**
- Show `t('readingPlans.sample')` with Ionicons `eye-outline`
- onPress: scroll the ScrollView to the day-1 entry section (use a ref)
- Style: same as Save For Later but with different icon

Layout: Row with `flexDirection: 'row'`, `gap: spacing.sm`, each button `flex: 1` (equal width) or Start Plan takes more space (`flex: 2`) and the other two share (`flex: 1` each).

- Style: `typography.body`, `colors.secondaryText`, `marginVertical: spacing.lg`, `paddingHorizontal: spacing.lg`

**Existing day-list:**

**Related Plans section:**
- Below the day-list, add a "Related Plans" section
- Header: `t('readingPlans.relatedPlans')` in `typography.sectionTitle` with `marginBottom: spacing.md`
- Query: call `getPlansByCategory(plan.category)` on mount, filter out the current plan by ID, limit to 5
- Render as a horizontal `FlatList`:
  - Each card: cover image (120x80, `borderRadius: radius.md`), plan title, duration days
  - Card is `TouchableOpacity` — onPress navigates to PlanDetail with the new planId:
    ```typescript
    navigation.push('PlanDetail', { planId: relatedPlan.id });
    ```
    Use `navigation.push` (not `navigate`) so it pushes a new screen on the stack rather than replacing the current one.
- If no related plans (empty category or only the current plan): hide the entire section
- Add bottom padding (`spacing.xxxl`) after Related Plans for scroll breathing room

**Image fallback pattern (reuse across all plan images in this file):**
```typescript
function PlanCoverImage({ uri, width, height, borderRadius }: {
  uri: string | null; width: number; height: number; borderRadius: number;
}) {
  const { colors } = useTheme();
  if (!uri) {
    return (
      <View style={{
        width, height, borderRadius,
        backgroundColor: colors.accentSecondary,
        justifyContent: 'center', alignItems: 'center',
      }}>
        <Ionicons name="book-outline" size={width * 0.3} color={colors.secondaryText} />
      </View>
    );
  }
  return (
    <Image
      source={{ uri }}
      style={{ width, height, borderRadius }}
      resizeMode="cover"
    />
  );
}
```

**Styling rules:**
- All colors from `useTheme()` — no hardcoded hex (the only exception: `rgba(0,0,0,0.4)` for the image gradient overlay is acceptable since it is a translucent black for contrast)
- All text from `t()` — no hardcoded user-facing strings
- Use `spacing`, `typography`, `radius` from `src/design/system.ts`
- Use `StyleSheet.create()` at component bottom
