# Plan 18-03: PlansHomeScreen Full Implementation

## Goal

When this plan is complete, the PlansHomeScreen has 4 fully functional sections behind the segmented control: My Plans (enrolled plans with progress bars + compact 14-day activity streak strip), Find Plans (featured hero card + topic category chips + sectioned plan cards), Saved (saved-for-later plans), and Completed (finished plans). Users can tap any plan card to navigate to PlanDetail.

## Scope

This plan does:
- Replace the placeholder content in `src/screens/plans/PlansHomeScreen.tsx` with full implementation
- Build a `MyPlansSection` sub-component (enrolled plans with cover thumbnail, progress bar, day counter)
- Build a compact 14-day activity streak strip component that uses `summarizeReadingActivity` from `src/services/progress/readingActivity.ts`
- Build a `FindPlansSection` sub-component (featured hero card, topic category chips grid, plan cards by category)
- Build a `SavedPlansSection` sub-component (list of saved plans with unsave action)
- Build a `CompletedPlansSection` sub-component (list of completed plans with completion date)
- Load all data on mount (all 4 sections cached in state), with pull-to-refresh
- Navigation: tapping any plan card calls `navigation.navigate('PlanDetail', { planId })`

This plan does NOT:
- Modify the PlanDetailScreen (that is Plan 18-04)
- Touch database migrations or service functions (Plan 18-01)
- Modify navigation wiring (Plan 18-02)
- Implement star ratings, audio plans, group assignment UI, or push reminders (deferred)

## Prerequisites

- Plan 18-01 must be complete (service functions `getFeaturedPlans`, `getSavedPlans`, `getCompletedPlans`, `getPlansByCategory`, and updated `ReadingPlan` type with `cover_image_url`, `featured`, `completion_count`)
- Plan 18-02 must be complete (PlansStack, PlansHomeScreen placeholder, i18n keys, navigation types)

## Tasks

### Task 1: Build PlansHomeScreen with data loading and segmented control

Rewrite `src/screens/plans/PlansHomeScreen.tsx` as a full implementation. The structure:

```
SafeAreaView
  Screen title: t('readingPlans.title')
  Segmented control (horizontal ScrollView of pill buttons)
  Content area (switches by activeTab):
    'my-plans'   -> <MyPlansSection />
    'find-plans' -> <FindPlansSection />
    'saved'      -> <SavedPlansSection />
    'completed'  -> <CompletedPlansSection />
```

**Data loading strategy:**
- On mount, fire all data fetches in parallel:
  - `listReadingPlans()` -> all plans (used by Find Plans section)
  - `getUserPlanProgress()` -> all enrolled progress rows (used by My Plans section)
  - `getSavedPlans()` -> saved plan list
  - `getCompletedPlans()` -> completed plan list
  - `getFeaturedPlans()` -> featured plan(s) for hero
- Store in component state: `allPlans`, `userProgress`, `savedPlans`, `completedPlans`, `featuredPlans`
- Show `ActivityIndicator` centered while loading. No loading skeleton on tab switches (data is already cached).
- `RefreshControl` on the outer ScrollView for pull-to-refresh.

**Segmented control** (already scaffolded in placeholder):
- Horizontal `ScrollView` of `TouchableOpacity` pill buttons
- Active: `backgroundColor: colors.accentPrimary`, text `colors.cardBackground`
- Inactive: `borderWidth: 1`, `borderColor: colors.cardBorder`, text `colors.secondaryText`
- Pill style: `paddingHorizontal: spacing.lg`, `paddingVertical: spacing.sm`, `borderRadius: radius.pill`

**Navigation:** Use `useNavigation()` with `NativeStackNavigationProp<PlansStackParamList>` to call `navigation.navigate('PlanDetail', { planId })`.

**All sub-components defined in the same file** (PlansHomeScreen.tsx) to keep imports simple. If the file exceeds ~500 lines, extract sub-components into `src/screens/plans/components/` with barrel export. Use Claude's judgment.

### Task 2: Build My Plans section with compact streak strip

**MyPlansSection** receives: `allPlans: ReadingPlan[]`, `userProgress: UserReadingPlanProgress[]`, `onPlanPress: (planId: string) => void`

**Enrolled plan cards:**
- For each progress row where `is_completed === false`, find matching plan from `allPlans`
- Render as a horizontal card: cover image thumbnail (60x60, rounded `radius.md`, `Image` with `cover_image_url` or fallback solid color), plan title (from `t(plan.title_key)`), day counter `t('readingPlans.dayOf', { current: progress.current_day, total: plan.duration_days })`, progress bar (same `ProgressBar` pattern as existing `ReadingPlanListScreen`)
- Wrap in `TouchableOpacity` calling `onPlanPress(plan.id)`
- If no enrolled plans, show `t('readingPlans.noActivePlans')` empty state text

**Compact 14-day activity streak strip:**
- Positioned below the enrolled plans list
- Shows last 14 days as a row of small circles (diameter 28, gap spacing.xs)
- For each day: filled circle (colors.accentPrimary) if reading activity exists, hollow circle (colors.cardBorder border) if no activity
- Use `summarizeReadingActivity` from `src/services/progress/readingActivity.ts` to get `daysByDateKey` map
- Read `chaptersRead` from the progress store or reading history service (use the same data source as `ReadingActivityScreen` in More tab)
- Use `formatLocalDateKey` from `readingActivity.ts` to generate date keys for last 14 days
- Below the dots row: "X day streak" label using the `currentStreak` from `summarizeReadingActivity`

Data source for reading activity: Import `useProgressStore` from `src/stores/progressStore` to get `chaptersRead` (the same `Record<string, number>` that `ReadingActivityScreen` uses). If `progressStore` does not expose `chaptersRead` directly, check the store and the `ReadingActivityScreen` implementation to identify the correct accessor. The goal is to reuse the exact same data pipeline.

### Task 3: Build Find Plans section with hero and category chips

**FindPlansSection** receives: `allPlans: ReadingPlan[]`, `featuredPlans: ReadingPlan[]`, `onPlanPress: (planId: string) => void`

**Featured hero card:**
- If `featuredPlans.length > 0`, render first featured plan as a large card:
  - Full-width `Image` (height 200, `borderRadius: radius.lg`) with `cover_image_url`
  - Overlay at bottom-left: duration badge `"{plan.duration_days}-DAY PLAN"` in a semi-transparent dark pill
  - Below image: plan title (typography.sectionTitle), plan description (first 100 chars of `t(plan.description_key)`)
  - Whole card is `TouchableOpacity` calling `onPlanPress`
- If no featured plans, skip hero section

**Topic category chips:**
- A grid of colored chips for plan discovery
- Categories to display (hardcoded labels, each maps to a filter):
  - Love, Healing, Hope, Anxiety, Anger, Depression, Faith, Prayer
- Render as a 2-row horizontal `ScrollView` or a `flexWrap` grid (Claude's discretion based on what fits better)
- Each chip: colored background from a small palette array (use theme-compatible muted tones), white text, rounded pill
- Tapping a chip is a no-op for now (future: filter by tag). Add a `// TODO: filter by tag` comment.

**Plan cards by category:**
- Group `allPlans` by `category` field
- For each category that has plans, render a section:
  - Section header: category label (capitalize, e.g., "Book Study", "Chronological", "Devotional", "Topical")
  - Horizontal `FlatList` of plan cards
  - Each card: cover image (120x80, radius.md), plan title, duration badge, "Start" button (or "Enrolled" if already enrolled)
  - Cards are `TouchableOpacity` calling `onPlanPress`

### Task 4: Build Saved and Completed sections

**SavedPlansSection** receives: `savedPlans: ReadingPlan[]`, `onPlanPress: (planId: string) => void`, `onRefresh: () => void`

- If `savedPlans.length === 0`: show empty state with `t('readingPlans.noSavedPlans')` centered text
- Otherwise: `FlatList` of plan cards (same card style as My Plans but without progress bar)
- Each card: cover thumbnail, plan title, duration, category badge
- Tapping navigates to PlanDetail

**CompletedPlansSection** receives: `completedPlans: Array<UserReadingPlanProgress & { plan: ReadingPlan }>`, `onPlanPress: (planId: string) => void`

- If empty: show `t('readingPlans.noCompletedPlans')` centered text
- Otherwise: `FlatList` of completed plan cards
- Each card: cover thumbnail, plan title, completion checkmark icon (Ionicons `checkmark-circle`, colors.accentPrimary), completed date (formatted from `completed_at`)
- Tapping navigates to PlanDetail

**Important styling rules:**
- All colors from `useTheme()` — no hardcoded hex values
- All text from `t()` — no hardcoded strings
- Use `spacing`, `typography`, `radius` from `src/design/system.ts`
- Use `StyleSheet.create()` at component bottom, not inline styles
- Image fallback: if `cover_image_url` is null, render a `View` with `backgroundColor: colors.accentSecondary` and centered Ionicons `book-outline` icon
