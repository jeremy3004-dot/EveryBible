# Plan 18-01: Database Migration, Type Updates, and Service Additions

## Goal

When this plan is complete, the Supabase schema has `cover_image_url`, `featured`, and `completion_count` columns on `reading_plans`, a new `user_saved_plans` table with RLS, the TypeScript types reflect these additions, the service layer has save/unsave/getSaved/getCompleted/getFeatured/getPlansByCategory functions, and all 8 reading plans have their day-by-day entry rows seeded in `reading_plan_entries`.

## Scope

This plan does:
- Create a Supabase migration file that ALTERs `reading_plans` and CREATEs `user_saved_plans`
- Update `cover_image_url`, `featured`, and `completion_count` for all 8 existing plans in the same migration
- Create a second migration file that seeds all ~978 `reading_plan_entries` rows for the 8 plans
- Add `cover_image_url`, `featured`, `completion_count` fields to the `ReadingPlan` interface in `src/services/supabase/types.ts`
- Add a new `UserSavedPlan` interface to `src/services/supabase/types.ts`
- Add `user_saved_plans` to the `Database['public']['Tables']` type map in `src/services/supabase/types.ts`
- Add 6 new service functions to `src/services/plans/readingPlanService.ts`: `savePlanForLater`, `unsavePlan`, `getSavedPlans`, `getCompletedPlans`, `getFeaturedPlans`, `getPlansByCategory`
- Export the new functions from `src/services/plans/index.ts`

This plan does NOT:
- Touch any screen files, navigation files, or i18n files
- Create the PlansStack or modify TabNavigator
- Build any UI components

## Prerequisites

None. This is the foundation plan with no dependencies.

## Tasks

### Task 1: Create Supabase migration for schema additions

Create file: `supabase/migrations/20260407000100_reading_plans_v2.sql`

This migration must:

1. ALTER TABLE `reading_plans` to add three columns:
   ```sql
   ALTER TABLE reading_plans ADD COLUMN IF NOT EXISTS cover_image_url TEXT;
   ALTER TABLE reading_plans ADD COLUMN IF NOT EXISTS featured BOOLEAN DEFAULT false;
   ALTER TABLE reading_plans ADD COLUMN IF NOT EXISTS completion_count INTEGER DEFAULT 0;
   ```

2. UPDATE each of the 8 existing plans with `cover_image_url` and `featured` values:
   - `bible-in-1-year`: featured = true, cover_image_url = `https://picsum.photos/seed/bible1year/800/400`
   - `new-testament-90-days`: featured = false, cover_image_url = `https://picsum.photos/seed/nt90/800/400`
   - `psalms-30-days`: featured = false, cover_image_url = `https://picsum.photos/seed/psalms30/800/400`
   - `gospels-60-days`: featured = false, cover_image_url = `https://picsum.photos/seed/gospels60/800/400`
   - `proverbs-31-days`: featured = false, cover_image_url = `https://picsum.photos/seed/proverbs31/800/400`
   - `genesis-to-revelation-chronological`: featured = false, cover_image_url = `https://picsum.photos/seed/chronological/800/400`
   - `epistles-30-days`: featured = false, cover_image_url = `https://picsum.photos/seed/epistles30/800/400`
   - `sermon-on-the-mount-7-days`: featured = false, cover_image_url = `https://picsum.photos/seed/sermon7/800/400`
   - Set seed completion_count values: bible-in-1-year = 5280, chronological = 3150, psalms-30-days = 8420, gospels-60-days = 4100, nt-90-days = 2890, proverbs-31-days = 6730, epistles-30-days = 1950, sermon-on-the-mount = 12400

3. CREATE TABLE `user_saved_plans`:
   ```sql
   CREATE TABLE IF NOT EXISTS user_saved_plans (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
     plan_id UUID NOT NULL REFERENCES reading_plans(id) ON DELETE CASCADE,
     saved_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
     UNIQUE (user_id, plan_id)
   );
   CREATE INDEX idx_user_saved_plans_user ON user_saved_plans(user_id);
   ALTER TABLE user_saved_plans ENABLE ROW LEVEL SECURITY;
   ```

4. Add RLS policies on `user_saved_plans` (own-only for SELECT, INSERT, DELETE):
   ```sql
   CREATE POLICY "saved_plans_select_own" ON user_saved_plans
     FOR SELECT TO authenticated USING (user_id = (select auth.uid()));
   CREATE POLICY "saved_plans_insert_own" ON user_saved_plans
     FOR INSERT TO authenticated WITH CHECK (user_id = (select auth.uid()));
   CREATE POLICY "saved_plans_delete_own" ON user_saved_plans
     FOR DELETE TO authenticated USING (user_id = (select auth.uid()));
   ```

### Task 2: Create seed migration for reading plan entries

Create file: `supabase/migrations/20260407000200_seed_reading_plan_entries.sql`

This migration seeds `reading_plan_entries` rows for all 8 plans. Since the `reading_plans` table uses `gen_random_uuid()` for IDs, use subqueries to look up plan IDs by slug:

```sql
-- Helper: reference plan by slug
-- INSERT INTO reading_plan_entries (plan_id, day_number, book, chapter_start, chapter_end)
-- SELECT id, <day>, '<BOOK_ID>', <start>, <end_or_null> FROM reading_plans WHERE slug = '<slug>';
```

Use the app's 3-letter book IDs from `src/constants/books.ts`: GEN, EXO, LEV, NUM, DEU, JOS, JDG, RUT, 1SA, 2SA, 1KI, 2KI, 1CH, 2CH, EZR, NEH, EST, JOB, PSA, PRO, ECC, SNG, ISA, JER, LAM, EZK, DAN, HOS, JOL, AMO, OBA, JON, MIC, NAH, HAB, ZEP, HAG, ZEC, MAL, MAT, MRK, LUK, JHN, ACT, ROM, 1CO, 2CO, GAL, EPH, PHP, COL, 1TH, 2TH, 1TI, 2TI, TIT, PHM, HEB, JAS, 1PE, 2PE, 1JN, 2JN, 3JN, JUD, REV.

Generate entries for each plan:

**Sermon on the Mount (7 days):**
- Day 1: MAT 5 (chapter_start=5, chapter_end=null)
- Day 2: MAT 6
- Day 3: MAT 7
- Day 4: MAT 5 (re-read with focus)
- Day 5: MAT 6
- Day 6: MAT 7
- Day 7: MAT 5 (chapter_start=5, chapter_end=7)

**Proverbs 31 Days:**
- Day 1-31: PRO 1 through PRO 31 (one chapter per day, chapter_end=null)

**Psalms 30 Days:**
- Day 1: PSA 1-5, Day 2: PSA 6-10, ..., Day 30: PSA 146-150
- Each day reads 5 Psalms, so chapter_start and chapter_end define the range

**Epistles 30 Days:**
- Split Romans through Jude (87 chapters across epistles) into 30 daily chunks of ~3 chapters each
- ROM (16ch), 1CO (16ch), 2CO (13ch), GAL (6ch), EPH (6ch), PHP (4ch), COL (4ch), 1TH (5ch), 2TH (3ch), 1TI (6ch), 2TI (4ch), TIT (3ch), PHM (1ch), HEB (13ch), JAS (5ch), 1PE (5ch), 2PE (3ch), 1JN (5ch), 2JN (1ch), 3JN (1ch), JUD (1ch)
- Insert one row per book-section per day. A day spanning 2 books needs 2 rows (same day_number, different book).

**Gospels 60 Days:**
- MAT (28ch) + MRK (16ch) + LUK (24ch) + JHN (21ch) = 89 chapters over 60 days
- ~1.5 chapters/day. Some days get 1 chapter, some get 2. Sequence: all of MAT, then MRK, then LUK, then JHN.

**NT in 90 Days:**
- 260 NT chapters from MAT through REV, spread over 90 days (~2.9/day)
- Sequential: MAT through REV. Each day gets 2-3 chapters.

**Bible in 1 Year (365 days):**
- Sequential through all 1189 chapters (GEN 1 through REV 22), ~3.25 chapters/day
- Each day: chapter_start to chapter_end range within a single book, or multiple rows if spanning books
- OT books first (GEN through MAL), then NT (MAT through REV)

**Chronological Bible (365 days):**
- Follow the standard chronological reading order (events in historical sequence)
- Day 1: GEN 1-3, Day 2: GEN 4-7, etc.
- Use the well-known public-domain chronological plan sequence (M'Cheyne-derived or Grant Horner simplified chronological)
- The key difference from sequential: Job inserts after Genesis 11, Psalms interleave with 1/2 Samuel, prophets interleave with Kings/Chronicles

For the two 365-day plans, generate entries programmatically using a Node.js script at `scripts/generate-plan-entries.ts`. The script should:
1. Define chapter counts per book (all 66 books)
2. For `bible-in-1-year`: distribute 1189 chapters across 365 days (~3.25/day), outputting SQL INSERT statements
3. For `genesis-to-revelation-chronological`: use the standard public-domain chronological reading order
4. Write output to `supabase/migrations/20260407000200_seed_reading_plan_entries.sql`

The script runs once during development (`npx tsx scripts/generate-plan-entries.ts`) and commits the generated SQL. The migration file itself is pure SQL.

The smaller plans (7, 30, 31, 60, 90-day) can be hand-written directly in the SQL file since they are straightforward sequential patterns.

### Task 4: Add service functions to readingPlanService.ts

Append 6 new exported functions to `src/services/plans/readingPlanService.ts`, following the exact same pattern as the existing 9 functions (use `requireSignedInUser`, return `PlanServiceResult<T>`, handle `isSupabaseConfigured()` guard):

1. **`savePlanForLater(planId: string): Promise<PlanServiceResult<UserSavedPlan>>`**
   - Requires auth via `requireSignedInUser('save a reading plan')`
   - INSERT into `user_saved_plans` with `user_id` and `plan_id`
   - Use `.upsert()` with `onConflict: 'user_id,plan_id'` for idempotency
   - Import `UserSavedPlan` from `../supabase/types`

2. **`unsavePlan(planId: string): Promise<PlanServiceResult>`**
   - Requires auth
   - DELETE from `user_saved_plans` where `user_id` = current user AND `plan_id` = planId

3. **`getSavedPlans(): Promise<PlanServiceResult<ReadingPlan[]>>`**
   - Requires auth
   - SELECT from `user_saved_plans` WHERE user_id = current user, then fetch the corresponding `reading_plans` rows
   - Use a two-step query: first get saved plan_ids, then fetch plans by those ids
   - Return the `ReadingPlan[]` array (not UserSavedPlan)

4. **`getCompletedPlans(): Promise<PlanServiceResult<(UserReadingPlanProgress & { plan: ReadingPlan })[]>>`**
   - Requires auth
   - SELECT from `user_reading_plan_progress` WHERE user_id = current user AND is_completed = true
   - Join with reading_plans to get plan metadata (use `.select('*, reading_plans(*)')` or a two-step query)
   - Return progress rows enriched with plan data

5. **`getFeaturedPlans(): Promise<PlanServiceResult<ReadingPlan[]>>`**
   - No auth required (public data)
   - Guard with `isSupabaseConfigured()`
   - SELECT from `reading_plans` WHERE featured = true AND is_active = true
   - Order by sort_order ascending

6. **`getPlansByCategory(category: string): Promise<PlanServiceResult<ReadingPlan[]>>`**
   - No auth required (public data)
   - Guard with `isSupabaseConfigured()`
   - SELECT from `reading_plans` WHERE category = category AND is_active = true
   - Order by sort_order ascending

Then update `src/services/plans/index.ts` to export all 6 new functions and the `UserSavedPlan` type re-export.
